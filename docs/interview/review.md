### 主页装修

#### 1. 主页装修与微前端有什么区别？

主页装修开放化本质上是同一个页面上不同组件的开放。

首先在应用场景上，尽管微前端框架 qiankun 可以使用`loadMicroApp` 手动同时加载多个子应用，每个挂载到不同的 DOM 容器中​，但是主页装修需要：

- 现存二十多个模块，每个模块就需要一个入口点，而且后续还需要增加新的入口点。
- 模块间需要排序等操作、去重等操作，实现起来较为困难。

最后，在性能上：

- 微前端性能较差，加载子应用时要加载 HTML、静态资源，耗时长
- 主页装修是加载同一个页面里的不同子组件，配合离线包，性能上等同原生开发。

### SSO

#### SSO 项目是如何防御 XSRF 攻击的？

- 每个业务都有对应的 sid 及对应的域名白名单，302 之后只能跳转到对应的白名单域名上。
- 不同业务的登录态是相互隔离的。

实际的 XSRF 接口由业务方自行处理，常见防御方式，可见：[CSRF](/browser-env/security/csrf)

XSRF 业务接口不支持跨域访问

### 大模型平台

常见的实现大模型结果输出（打印机效果）的方案：

- 纯前端：定时器（setInterval/setTimeout）+ 文案拼接
- 服务端流式输出（如 SSE） + 前端拼接
- 使用第三方库（如 Typed.js/​​TypeIt​​）

比如腾讯元宝，即是采用 SSE 输出 Markdown 文本，前端拼接 Markdown 文本，并通过 Markdown 相关的库渲染出来
