# 使用 Devtools 排查性能问题

## Devtools 面板使用概述

- ⭐️⭐️⭐️⭐️⭐️ [MDN - 解决内存问题](https://developer.chrome.com/docs/devtools/memory-problems?hl=zh-cn)

| 面板                                       | 核心功能                                                                                                           | 使用场景                                                                                                                                                                                                                                                                                                                                      | 优点                                                                                     | 缺点                                                               |
| ------------------------------------------ | ------------------------------------------------------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------- | ------------------------------------------------------------------ |
| Chrome 任务管理器                          | 实时监控每个页面的内存占用、JavaScript 内存、CPU 占用等等信息                                                      | -                                                                                                                                                                                                                                                                                                                                             | -                                                                                        | -                                                                  |
| Performance 面板（勾选 Memory）            | 查看 JS 堆内存、文档、DOM 节点、监听器、GPU 内存 的变化情况                                                        | -                                                                                                                                                                                                                                                                                                                                             | -                                                                                        | -                                                                  |
| Memory 面板的 Heap snapshot 选项           | 给当前内存拍一张“全家福”。它会记录拍摄时刻所有**存活的 JavaScript 对象和 DOM 节点**。                              | - 排查内存泄漏：这是最常用的功能。通常做法是：操作前拍一张，操作后（比如打开弹窗再关闭）拍一张，对比两张快照的差异（Delta）。如果某个对象的数量或内存大小在操作后不降反升，说明可能泄漏了。<br><br>- 分析内存占用：查看哪些对象占用了最多的内存（Retained Size）。<br><br> - 可查看分离的 DOM 节点（没在 DOM 树里，但是仍有 JS 引用）内存泄漏 | **信息最全**，能看到对象之间的引用关系（谁引用了谁），能精准定位泄漏根源。               | 生成快照比较耗时，且只能看到“拍摄那一刻”的状态，无法看到过程。     |
| Memory 面板的 Allocations on timeline 选项 | 在时间轴上**实时记录内存分配的过程**。它能显示对象是在什么时候被创建的，以及它们是否还“活着”。                     | - 排查“瞬时”泄漏：比如你在循环或动画帧（requestAnimationFrame）中频繁创建对象，想知道这些对象是否被及时回收。<br><br> - 观察内存波动：配合用户操作（如反复点击按钮），观察蓝色的“存活对象”条是否随着操作结束而消失。如果操作结束了，还有大量蓝色条存在，说明这些本该销毁的对象泄漏了。                                                        | 能看到内存分配的“动态过程”，非常适合捕捉那些产生快、消失也快的内存问题。                 | 记录非常详细，性能开销大，开启后页面可能会变卡，不适合长时间录制。 |
| Memory 面板的 Allocation sampling 选项     | 通过“抽样调查”的方式，统计内存分配情况。它不会记录每一个对象，而是通过采样来估算哪些函数或代码行分配了最多的内存。 | - 分析长时间运行的应用：比如页面跑了几个小时，想知道这段时间内主要是哪部分代码在消耗内存。<br><br> - 定位内存大户：快速找到消耗内存最多的代码执行栈（Execution Stack）。 <br><br> - 可查看按函数粒度分配的内存                                                                                                                                | 对页面性能影响极小（Overhead 低），适合在生产环境或长时间运行的场景下使用。              | 数据是估算的，不够精确，无法像堆快照那样看到具体的对象引用链。     |
| Memory 面板的 Detached elements 选项       | 这是一个专门针对 DOM 节点的快捷工具。它会列出所有已经从 DOM 树中移除，但仍然被 JavaScript 变量引用的节点。         | **专门排查 DOM 泄漏**：比如你移除了一个 `<div>`，但忘了移除绑定在它上面的事件监听器，或者有一个全局变量还指着它。                                                                                                                                                                                                                             | 简单直接，不需要像堆快照那样去茫茫多的对象中筛选 Detached 关键字，直接就能看到问题节点。 | 功能单一，只能看 DOM 节点，无法分析普通的 JS 对象泄漏。            |

Memory 面板总结建议：

- 如果你发现页面**卡顿、崩溃**，或者怀疑有**内存泄漏**，首选 **Heap snapshot**（拍对比快照）。
- 如果你想知道**哪段代码一直在疯狂创建对象**，用 **Allocations on timeline**。
- 如果你只是想快速看看**有没有没解绑的 DOM 节点**，用 **Detached elements**。
- 如果你想分析**页面运行很久后的整体内存分布**，且不想让电脑太卡，用 **Allocation sampling**。
